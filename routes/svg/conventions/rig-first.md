# Rig-first 变形合同

> 做任何手脚、尾巴、触角、脸部边缘、衣物边缘等会变形的部件前，先读这一页。

## 核心规则

不要直接拖生产 path 的原始控制点来做变形动画。

会变形的部件先建或复用一个小 rig：

1. 给语义锚点命名：根部、接触点、弯折点、末端、固定边。
2. 先让 0% path 和母版逐字一致，再做中间帧。
3. 先锁定用户/维护者认可的终点姿态，再调 easing 和二级动作。
4. 中间形态由 rig 生成，不靠临时点编辑凑。
5. 除非端点本身就是动作目标，否则端点先锁住。

刚体 transform 仍然是合法工具。一个部件本来就该整体平移、旋转、统一缩放时，可以直接用 transform。错误做法是拿整体 transform 或手拖控制点替代 deform。

## Dense warp 要求

部件 path 里常常同时有曲线和直线。只采样 `C` 曲线、不处理 `L` 段的 warp，会在混合路径上崩。

做 path warp 时：

- 结构化解析 path data；
- 尽量支持 `M`、`L`、`C`、`Q` 和 closepath；
- 先 dense 采样中线，再映射轮廓；
- 保留 winding 和 fill 行为；
- 信任动画前，先比对生成的 0% 帧是否等于源 path。

## 图层和接触关系

变形经常需要 helper layer。把 helper 当作 rig 的显式组成部分：

- 接触阴影、遮挡片、局部暗边要单独命名；
- helper layer 要 clip 到拥有它的身体或父表面；
- 不要把坏掉的肢体整条藏到身体后面；
- 如果 helper 只在单个姿势里验证过，标记为 pose-specific；做过渡动画前重新实看。

## Transform-origin 归零

SVG 文件常带继承 CSS，例如 `transform-origin: center`。如果 rig 输出会写显式 `transform` 属性，先给被动画部件重置：

```css
transform-box: view-box;
transform-origin: 0 0;
```

先做这步再判断几何。错误 origin 会让正确 rig 看起来像飞走。

## 验收清单

锁定 rig 动画前检查：

- 0% 等于母版姿态；
- 100% 等于确认过的终点姿态；
- 循环或过渡播放至少 30 秒无漂移；
- 关节没有尖角、坍塌、鼓包；
- 重要识别标志仍然可见；
- 常见浏览器缩放和目标 runtime/browser 都看过。

同一个问题连续两轮还以同样方式失败，就先停下，不要继续补症状。记录当前文件、现象、已试方案和边界，再重新审 rig 选择。
