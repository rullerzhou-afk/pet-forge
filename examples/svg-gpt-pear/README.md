# GPT Pear SVG Demo

This demo uses a GPT-generated transparent PNG as the source image for the SVG route.

## Files

- `source.png`: transparent PNG source.
- `pear.svg`: vtracer output generated by `png2svg.py` (13 anonymous paths, solid fills, ~21 KB).
- `pear-gpt-5.5-pro.svg`: the same reference re-drawn directly by GPT-5.5 Pro as a structured SVG (15 named paths grouped into `plant-and-limbs` / `body` / `face`, gradients + blur + clipPath, ARIA `title`/`desc`, ~12 KB). The white background rect the model emitted has been removed, so the file is transparent — a desktop pet floats on the desktop.
- `idle.svg.html`: runnable idle animation using the generated SVG.

## Tool Trace vs Direct Model Authoring

`pear.svg` and `pear-gpt-5.5-pro.svg` are two SVGs of the same character from the same reference image, produced two different ways. vtracer traces color regions: it reproduces the silhouette but flattens fine detail (eye highlights, mouth, tongue) and leaves anonymous paths. GPT-5.5 Pro re-authors the artwork as a clean, layered, rig-ready SVG, but the result is the model's own interpretation rather than a pixel-faithful trace. The README has a side-by-side comparison table.

## Source Check

- Image size: 1254 x 1254
- Transparency: real RGBA alpha, transparent corners
- Visible alpha: hard edge only, no semi-transparent pixels
- Visible RGB colors before quantization: about 16k

## Reproduce

From the repository root:

```powershell
py -3.13 routes\svg\tools\png2svg\png2svg.py examples\svg-gpt-pear\source.png examples\svg-gpt-pear\pear.svg --preset apple-precise
```

This run produced 13 SVG paths and a 21 KB SVG file.

## Result Notes

This is a better source for vtracer than a screenshot with a checkerboard background. The true alpha channel prevents the background from being vectorized, and the 8-color quantization keeps path count low.

The source still contains soft shading, so the SVG output is an approximation of the original image rather than a hand-clean vector drawing. For production-quality SVG pets, use flatter colors or redraw the main shapes manually after tracing.
